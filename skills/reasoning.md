# Reasoning Skill

## Purpose
Improve decisions on complex tasks through targeted abstraction.

## Use When
- problem is ambiguous or high-impact
- multiple viable paths exist
- surface request may hide the root cause

## Method
1. identify the real problem
2. state assumptions and principles
3. generate 2-3 viable paths
4. compare tradeoffs briefly
5. choose one path with a falsifiable validation check

## Constraints
- skip for trivial one-step work
- stop once reasoning improves execution
- avoid open-ended abstraction

Reasoning must improve execution, not replace it.
