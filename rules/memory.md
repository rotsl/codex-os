# Memory Rules

Pre-task:
- read `memory/context.md`
- read `memory/decisions.md`

Post-task:
- update both files with compressed durable summaries
- remove stale entries and duplicates
- do not store raw logs or temporary scratch context

Semantics:
- `memory/context.md` = current state, constraints, goals, in-progress
- `memory/decisions.md` = durable decisions, reasons, reusable patterns

Only store information that improves future performance.
